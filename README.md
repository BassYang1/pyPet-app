# uni-app TS 多端工程架基础配置
- 整体技术栈: Vite 5 + Vue3.5 + TypeScript5.4 + uni-app 3.x

## 生产依赖 dependencies（项目运行必需）
1. **@dcloudio/uni-app**
   版本：3.0.0-4080720251210001
   用途：uni-app 3.x 核心框架，提供跨端基础生命周期、全局API、路由与基础组件能力，基于 Vite+Vue3 架构，是项目运行的底层基座。
   要点：全系 uni 官方包需保持版本一致，不可随意降级/升级，所有端运行必备。

2. **@dcloudio/uni-app-plus**
   版本：3.0.0-4080720251210001
   用途：App 端专属增强包，封装安卓、iOS 原生能力、原生插件与渲染层逻辑。
   要点：仅作用于 App 端，H5、小程序不生效，打包 App 必须依赖。

3. **@dcloudio/uni-h5**
   版本：3.0.0-4080720251210001
   用途：H5 端适配层，处理浏览器 DOM 兼容、H5 专属组件与环境 API。
   要点：运行、发布 H5 页面必备，需和主框架版本严格对齐。

4. **@dcloudio/uni-mp-alipay**
   版本：3.0.0-4080720251210001
   用途：支付宝小程序转译适配器，抹平 uni 语法与支付宝小程序原生语法、API 差异。
   要点：编译支付宝小程序时生效，属于多端适配专用依赖。

5. **@dcloudio/uni-mp-weixin**
   版本：3.0.0-4080720251210001
   用途：微信小程序适配层，负责语法转译、微信专属 API、页面渲染与分包逻辑。
   要点：编译微信小程序核心依赖，版本必须与 uni 主框架统一。

6. **pinia**
   版本：^2.1.7
   用途：Vue3 官方状态管理工具，用于全局数据、跨页面状态共享，替代传统 Vuex。
   要点：轻量简洁，对 TypeScript 友好，支持模块化与状态持久化，uni-app 3.x 推荐使用。

7. **vue**
   版本：^3.5.10
   用途：Vue3 核心框架，提供响应式、组合式 API、组件系统、模板编译能力。
   要点：uni-app 3.x 仅支持 Vue3，`^` 符号允许同主版本内小版本自动更新。

## 开发依赖 devDependencies（仅本地编译/开发使用，不打包进线上代码）
1. **@dcloudio/types**
   版本：3.4.19
   用途：uni-app 官方 TypeScript 类型声明文件，为 uni 全局 API、内置组件提供代码提示与类型校验。
   要点：TS 项目必备，解决开发阶段类型缺失、无代码提示问题。

2. **@dcloudio/vite-plugin-uni**
   版本：3.0.0-4080720251210001
   用途：Vite 适配 uni-app 的核心编译插件，负责页面转译、条件编译、多端打包、热更新。
   要点：工程核心插件，**必须和所有 @dcloudio 包版本完全一致**，版本不一致会直接编译报错。

3. **@types/node**
   版本：^22.19.21
   用途：Node.js 内置 API 的 TS 类型声明。
   要点：开发中使用 Node 脚本、配置文件时提供类型提示，仅开发环境生效。

4. **@uni-helper/uni-app-types**
   版本：^1.0.1
   用途：uni-app 增强类型补充包，补齐官方类型缺失的组件属性、事件、页面配置等类型。
   要点：优化 TS 开发体验，减少 `any` 类型，适配 uni-ui 等扩展组件。

5. **@vitejs/plugin-vue**
   版本：^4.6.2
   用途：Vite 解析 Vue 单文件组件（.vue）的官方插件。
   要点：Vite + Vue3 标配依赖，负责解析模板、脚本、样式代码。

6. **sass**
   版本：^1.101.0
   用途：SCSS/Sass 样式预处理器，支持样式变量、嵌套、混合器等高级写法。
   要点：项目中 `<style lang="scss">` 语法依赖，为新版 Dart Sass，无老旧 node-sass 兼容问题。

7. **typescript**
   版本：^5.4.5
   用途：TypeScript 核心编译器，实现静态类型检查、TS 语法编译。
   要点：全项目类型约束核心，搭配 Vue3、Pinia 实现类型安全。

8. **unplugin-auto-import**
   版本：^0.16.7
   用途：API 自动导入插件，无需手动 import 即可使用 ref、reactive、uni 等常用方法。
   要点：减少重复代码，提升开发效率，搭配 TS 需额外配置类型生成。

9. **unplugin-vue-components**
   版本：^0.25.2
   用途：组件自动导入插件，自动识别并引入项目内自定义组件、第三方组件库。
   要点：无需页面手动引入组件，是 Vite 项目常用优化方案。

10. **vite**
    版本：^5.2.8
    用途：项目核心构建工具，负责项目启动、热更新、代码编译、多端打包。
    要点：uni-app 3.x 默认构建工具，启动与打包速度优异，要求 Node 版本 ≥ 18.x。

11. **vue-tsc**
    版本：^2.0.16
    用途：Vue 单文件组件专属 TS 类型检查工具。
    要点：弥补原生 tsc 无法完整校验 .vue 文件的问题，常用于打包、CI 流程做全量类型检测。

## 补充说明
1. 所有 `@dcloudio` 开头官方包版本完全统一，是项目正常编译运行的硬性要求。
2. 版本前 `^` 代表：锁定主版本，允许次版本、补丁版本自动升级。
